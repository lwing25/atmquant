# 小时交易时段修正总结

## 修正时间
2025-10-06

## 修正原则

### 1. 核心规则
- **时段范围**：闭区间 `[start, end]`，表示该时段包含的所有分钟
- **时段长度**：每个小时时段尽量接近60分钟，而非简单按上午盘/下午盘划分
- **结束时间**：时段的end时间应该是**最后一根1分钟K线的时间戳**
  - 例如：02:30结束的时段，最后一根K线是02:29
  - 时间戳02:29表示02:29:00-02:29:59的数据

### 2. 时段划分策略
- **连续交易市场**（如美股、欧股）：按自然小时划分
- **有休盘市场**（如中国、香港、日本）：考虑休盘时间，合理划分时段
- **目标**：每个时段长度接近1小时，便于小时K线聚合分析

### 1. 中国期货市场（CN_FUTURES_SESSION）

**问题**：
- 夜盘最后一个时段 `(time(2, 0), time(2, 30))` 包含31分钟（02:00-02:30）

**修正**：

(time(2, 0), time(2, 30))  # ❌ 包含02:00-02:30，共31分钟

(time(2, 0), time(2, 29))  # ✅ 包含02:00-02:29，共30分钟

### 2. 港股主板（HK_STOCK_SESSION）

**问题**：
- 原设置将上午盘和下午盘整体划分，不利于小时K线分析
  - 上午盘：`(time(9, 30), time(11, 59))` - 2.5小时
  - 下午盘：`(time(13, 0), time(15, 59))` - 3小时

**修正**：拆分为接近1小时的时段
```python

    (time(9, 30), time(11, 59)),     # ❌ 上午盘2.5小时
    (time(13, 0), time(15, 59))      # ❌ 下午盘3小时
]

# 修正后
hour_sessions=[
    (time(9, 30), time(10, 29)),     # ✅ 第1小时：60分钟
    (time(10, 30), time(11, 29)),    # ✅ 第2小时：60分钟
    (time(11, 30), time(11, 59)),    # ✅ 第3时段：30分钟（上午盘最后）
    (time(13, 0), time(13, 59)),     # ✅ 第4小时：60分钟
    (time(14, 0), time(14, 59)),     # ✅ 第5小时：60分钟
    (time(15, 0), time(15, 59))      # ✅ 第6小时：60分钟
]
```

**实际交易时间**：09:30-12:00, 13:00-16:00

## 设计考虑

### 1. 为什么不是所有时段都是60分钟？

**原因**：
- 某些市场的交易时间不是整小时的倍数
- 例如：港股上午盘 09:30-12:00 = 2.5小时
- 我们采用的策略：
  - 优先划分为60分钟的完整小时
  - 剩余时间单独作为一个时段

### 2. 休盘时间如何处理？

**策略**：
- **短休盘**（≤15分钟）：合并到同一时段
  - 例如：中国期货10:15-10:30休盘，合并为10:00-11:14
- **长休盘**（>15分钟）：分为不同时段
  - 例如：港股12:00-13:00午休，分为上午盘和下午盘

### 3. 时段起止时间如何确定？

**原则**：
- **起始时间**：该时段第一根1分钟K线的时间戳
- **结束时间**：该时段最后一根1分钟K线的时间戳
- **时间戳含义**：time(9, 30) 表示 09:30:00-09:30:59 的数据

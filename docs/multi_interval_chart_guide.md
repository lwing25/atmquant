# 多周期图表使用指南

## 概述

EnhancedChartWidget 现已支持多周期图表切换功能，允许用户在同一个图表中查看不同时间周期的K线数据。系统会自动将1分钟K线数据聚合为更大的时间周期。

## 功能特性

### 支持的时间周期

- **1分钟**：显示原始1分钟K线数据
- **5分钟**：将5根1分钟K线聚合为1根5分钟K线
- **15分钟**：将15根1分钟K线聚合为1根15分钟K线
- **1小时**：将60根1分钟K线聚合为1根1小时K线
- **日线**：将一天内的所有1分钟K线聚合为1根日K线

### 界面设计

时间周期切换面板位于图表左侧，采用以下设计：

- **位置**：窗口左侧，垂直居中显示
- **透明度**：半透明背景（rgba(30, 30, 30, 150)），不影响K线图表查看
- **文字排列**：竖排显示，节省空间，美观大方
- **按钮样式**：
  - 未选中：灰色背景
  - 鼠标悬停：浅灰色高亮
  - 选中状态：蓝色高亮显示
- **响应式**：窗口调整大小时，面板自动保持垂直居中

### K线聚合规则

聚合规则遵循标准的K线合成原则：

- **开盘价**：使用周期内第一根K线的开盘价
- **最高价**：使用周期内所有K线的最高价
- **最低价**：使用周期内所有K线的最低价
- **收盘价**：使用周期内最后一根K线的收盘价
- **成交量**：周期内所有K线成交量的总和
- **成交额**：周期内所有K线成交额的总和
- **持仓量**：使用周期内最后一根K线的持仓量

## 使用方法

### 1. 基本使用

```python
from vnpy.trader.ui import create_qapp
from vnpy.trader.database import get_database
from vnpy.trader.constant import Exchange, Interval
from core.charts.enhanced_chart_widget import EnhancedChartWidget

# 创建应用
qapp = create_qapp()

# 创建图表
chart = EnhancedChartWidget()
chart.resize(1400, 800)
chart.show()

# 加载1分钟数据
database = get_database()
bars = database.load_bar_data(
    symbol="rb2501",
    exchange=Exchange.SHFE,
    interval=Interval.MINUTE,
    start=start_date,
    end=end_date
)

# 更新图表（会自动保存1分钟数据作为基础数据）
chart.update_history(bars)

# 运行应用
qapp.exec()
```

### 2. 周期切换

图表打开后，可以通过以下方式切换周期：

1. 点击左侧面板中的任意周期按钮
2. 系统会自动将1分钟数据聚合为目标周期
3. 所有技术指标会基于新周期的K线重新计算
4. 图表会自动更新显示

### 3. 注意事项

**数据要求：**
- 必须先加载1分钟K线数据
- 系统会自动保存1分钟数据作为基础数据
- 切换周期时不会重新从数据库加载，而是直接聚合已有的1分钟数据

**聚合行为：**
- 日线聚合：按交易日期进行聚合
- 其他周期：按时间段进行聚合（例如9:00-9:04的5分钟聚合为一根K线）
- 非交易时间的数据会被正确处理

**指标计算：**
- 切换周期后，所有技术指标会自动重新计算
- 指标参数保持不变，但计算基于新周期的K线
- 例如：MA(5)在1分钟图是5分钟均线，在5分钟图是25分钟均线

## 技术实现

### 核心方法

**`_aggregate_bars(minute_bars, target_interval)`**
- 将1分钟K线聚合为目标周期
- 参数：
  - `minute_bars`: 1分钟K线数据列表
  - `target_interval`: 目标周期（5m, 15m, 1h, d）
- 返回：聚合后的K线数据列表

**`_on_interval_changed(interval)`**
- 处理周期切换事件
- 更新按钮激活状态
- 重新聚合数据并更新图表

**`update_history(history)`**
- 重写了父类方法
- 自动保存1分钟数据作为基础数据
- 根据当前周期决定是否需要聚合

### 数据流程

```
加载1分钟数据
    ↓
保存到 base_minute_bars
    ↓
用户点击周期按钮
    ↓
调用 _on_interval_changed
    ↓
调用 _aggregate_bars 聚合数据
    ↓
调用父类 update_history 更新图表
    ↓
图表自动重新计算指标
```

## 扩展功能

### 添加新的时间周期

如果需要添加新的时间周期，可以在 `_create_interval_panel` 方法中修改：

```python
intervals = [
    ("1m", "1分钟", Interval.MINUTE),
    ("5m", "5分钟", "5m"),
    ("15m", "15分钟", "15m"),
    ("30m", "30分钟", "30m"),  # 新增30分钟
    ("1h", "1小时", Interval.HOUR),
    ("d", "日线", Interval.DAILY),
]
```

同时需要在 `_aggregate_bars` 方法中添加对应的聚合规则：

```python
interval_minutes = {
    "1m": 1,
    "5m": 5,
    "15m": 15,
    "30m": 30,  # 新增
    "1h": 60,
    "d": 1440
}
```

## 常见问题

### Q: 为什么切换周期后K线数量变少了？

A: 这是正常的。周期越大，K线数量越少。例如：
- 1分钟周期：1000根K线
- 5分钟周期：约200根K线（1000÷5）
- 1小时周期：约17根K线（1000÷60）

### Q: 为什么日线图只有几根K线？

A: 日线图是将所有日内数据聚合为一根K线。如果你的数据只有5个交易日，那么日线图就只有5根K线。

### Q: 可以直接从数据库加载5分钟、15分钟的数据吗？

A: 不需要。系统会自动从1分钟数据聚合，这样可以：
- 减少数据库存储空间
- 确保数据一致性
- 简化数据管理

### Q: 切换周期会影响指标显示吗？

A: 会的。指标会基于新周期的K线重新计算。例如：
- 1分钟图的MA(20)是20分钟均线
- 5分钟图的MA(20)是100分钟均线
- 日线图的MA(20)是20日均线

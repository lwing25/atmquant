# 双图并排显示功能演示

## 快速开始

### 在回测界面使用

1. **打开K线图表**

   运行回测后，点击"K线图表"按钮：

   ![打开图表](images/open_chart.png)

2. **切换到双图模式**

   点击左上角的"双图模式"按钮：

   ```
   [单图模式] → 点击 → [双图模式 ✓]
   ```

   界面会自动切换为左右两个图表显示

3. **调整周期**

   每个图表顶部都有周期选择面板：

   - 左侧图表：可选择 1m / 5m / **15m** / 1h / d
   - 右侧图表：可选择 1m / 5m / 15m / **1h** / d

   （默认左侧15分钟，右侧1小时）

4. **启用时间轴同步**

   点击顶部的"启用同步"按钮：

   ```
   [启用同步] → 点击 → [同步已启用 ✓]
   ```

   启用后：
   - 在左侧图表拖拽/缩放 → 右侧图表自动跟随
   - 在右侧图表拖拽/缩放 → 左侧图表自动跟随

5. **调整显示比例**

   拖动中间的分割线可以调整左右图表的显示比例：

   ```
   [=====|=====]  默认1:1
   [===|=======]  左小右大
   [=======|===]  左大右小
   ```

## 功能特性

### ✨ 核心功能

| 功能 | 说明 |
|-----|-----|
| 双图显示 | 左右并排显示两个图表 |
| 独立周期 | 每个图表可独立选择周期 |
| 时间同步 | X轴时间范围自动对齐 |
| 比例调整 | 可拖拽调整显示比例 |
| 模式切换 | 单图/双图一键切换 |

### 📊 支持周期

- **1m** - 1分钟
- **5m** - 5分钟
- **15m** - 15分钟 (左侧默认)
- **1h** - 1小时 (右侧默认)
- **d** - 日线

### 🔄 同步机制

```
启用同步后：

左侧15分钟图               右侧1小时图
┌─────────────┐           ┌─────────────┐
│  [时间范围] │ ←同步→    │  [时间范围] │
│   9:00 ─    │           │   9:00 ─    │
│   11:00     │           │   11:00     │
└─────────────┘           └─────────────┘

拖拽/缩放其中一个 → 另一个自动跟随
```

## 操作技巧

### 💡 最佳实践

1. **数据准备**
   - 使用1分钟K线作为基础数据
   - 确保数据完整连续
   - 数据量建议在10000根以内

2. **周期搭配建议**

   | 左侧周期 | 右侧周期 | 适用场景 |
   |---------|---------|---------|
   | 1m | 5m | 超短线交易 |
   | 5m | 15m | 短线交易 |
   | 15m | 1h | 日内交易 |
   | 1h | d | 波段交易 |

3. **性能优化**
   - 大数据量时关闭部分指标
   - 避免频繁切换周期
   - 先加载数据再启用同步

### ⚡ 快捷操作

```
[Ctrl + 拖拽]  - 在图表上框选放大区域
[双击Y轴区域]  - 重置Y轴范围
[滚轮]        - 缩放X轴范围
[拖拽]        - 平移图表
```

## 常见问题

### Q1: 为什么同步后两个图表显示的K线数量不同？

**A**: 这是正常的。不同周期的K线数量本来就不同。例如：
- 1小时包含60根1分钟K线
- 同步的是时间范围，不是K线数量

### Q2: 如何调整图表显示比例？

**A**: 拖动中间的分割线即可调整左右图表的显示比例。

### Q3: 切换周期后交易信号消失了？

**A**: 目前交易信号显示仅支持单图模式。双图模式下暂不显示交易信号。

### Q4: 可以同时显示3个或更多图表吗？

**A**: 当前版本仅支持双图。3图/4图功能计划在后续版本中实现。

### Q5: 为什么有时候同步不准确？

**A**: 可能是因为：
- 不同周期K线时间点不完全对齐
- 系统使用最近匹配算法，会找最接近的K线
- 这是正常的，不影响分析
